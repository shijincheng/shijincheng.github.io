必须掌握的特性

配置文件管理

容器配置管理的需求：

  1. 一致性（用不用容器这都是个硬需求）模板化（同上）
  2. 尽可能在配置变更时不重启容器
  3. 配置文件与容器的强绑定（容器销毁重建时，其配置文件应该自动跟随）
  4. 配置参数动态获取（如：数据库的域名、端口）
  5. 需要保证容器创建之后、启动之前配置文件已经Ready
  6. 易用性（包括配置变更成本、配置文件管理系统本身的维护成本）

参考：https://zhuanlan.zhihu.com/p/27700368

配置文件集中管理的两个方向

  1. 推
  2. 拉

    推

    应用启动时主动拉取配置文件，启动后配置文件管理服务检测到配置发生变化，由配置文件管理服务向容器推送配置文件。
    可以实现推送方式的工具：
  • zookeeper
  • inotify + rsync
  • git + ansible

    拉

    应用启动时主动拉取配置文件，启动后定时任务拉取配置文件。
    可以实现拉取方式的工具：
  • git
  • vault
